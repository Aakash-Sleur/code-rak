  # Frontend Development Standards

  ## Technology Stack
  - **Framework**: Next.js 16 (React 19)
  - **Styling**: Tailwind CSS v4 with OKLch color model
  - **UI Components**: shadcn/ui built on BaseUI primitives
  - **Icons**: Tabler Icons (@tabler/icons-react v3.41.1)
  - **Authentication**: NextAuth v4
  - **Editor**: Monaco Editor (@monaco-editor/react)
  - **Utilities**: clsx, tailwind-merge, class-variance-authority
  - **Notifications**: Sonner (toast notifications)

  ## Styling Approach

  ### Color Scheme (OKLch-based)
  The application uses an OKLch perceptually uniform color space:
  - **Primary**: `oklch(0.216 0.006 56.043)` (dark gray-blue)
  - **Destructive**: `oklch(0.577 0.245 27.325)` (red)
  - **Background**: `oklch(1 0 0)` (pure white)
  - **Foreground**: `oklch(0.147 0.004 49.25)` (near black)
  - **Muted**: `oklch(0.97 0.001 106.424)` (light gray)
  - **Border**: `oklch(0.923 0.003 48.717)` (light border)
  - **Ring/Focus**: `oklch(0.709 0.01 56.259)` (medium blue-gray)

  Use CSS custom properties (`--color-*`, `--color-*-foreground`) defined in `/app/globals.css`.

  ### Tailwind CSS Conventions
  - Use Tailwind utility classes exclusively for styling
  - Apply `prettier-plugin-tailwindcss` for class ordering
  - Chain with `cn()` utility from `/lib/utils.ts` for conditional styles
  - Prefer semantic utility combinations over custom CSS
  - Use `data-*` attributes for state-based styling (data-open, data-disabled, etc.)
  - Border radius is set to 0 by default; use explicit radius utilities if needed

  ### Example Tailwind Usage
  ```tsx
  className="flex items-center gap-2 rounded-lg bg-primary px-4 py-2 text-primary-foreground hover:bg-primary/90 transition-colors"
  ```

  Preferred components from shadcn/ui built on BaseUI:
  - `Button` - Always use `variant` prop (default, outline, ghost, destructive)
  - `Card` - With CardHeader, CardContent, CardDescription, CardTitle
  - `Dialog` - Composable: Dialog, DialogTrigger, DialogContent, DialogOverlay
  - `Dropdown Menu` - Composable: DropdownMenu, DropdownMenuTrigger, DropdownMenuContent, DropdownMenuItem, DropdownMenuSeparator
  - `Input` - For text inputs with built-in sizing
  - `Avatar` - With AvatarFallback
  - `Badge` - For status indicators
  - `Checkbox` - Styled with Tabler Icons
  - `Label` - Form labels
  - `Alert` - For alerts and notifications
  - `Tooltip` - For help text
  - `Pagination` - For paginated lists
  - `ScrollArea` - For scrollable content
  - `Separator` - Visual dividers
  - `Sheet` - Side sheet drawer
  - `Skeleton` - Loading placeholders
  - `InputOTP` - OTP input component
  - `Sonner` - Toast notifications

  Location: `/components/ui/`

  ### BaseUI Integration
  All components use BaseUI as the unstyled primitive layer:
  - `@base-ui/react/menu` - For DropdownMenu (Menu, Popup, Positioner, Portal, Trigger)
  - `@base-ui/react/dialog` - For Dialog (Root, Trigger, Backdrop, Popup, Portal, Close)
  - `@base-ui/react/tabs` - For tabbed interfaces
  - `@base-ui/react/tooltip` - For tooltips
  - `@base-ui/react/scroll-area` - For scrollable regions

  Reference BaseUI docs: https://base-ui.com/

  #### CustomAlert
  Alert component with 5 built-in types and icons:
  ```tsx

    type="success" // "default" | "success" | "warning" | "destructive" | "info"
    title="Alert Title"

  />
  ```
  - Types: `default`, `success` (green), `warning` (yellow), `destructive` (red), `info` (blue)
  - Automatic icon selection based on type
  - Optional action button
  - Full Tabler Icons integration
  
  #### Notification/Toast System
  Sonner-based toast notifications with helper function:
  ```tsx
  import { showNotification } from "@/components/custom/custom-notification"
  
  showNotification("Success!", {
    type: "success", // "default" | "success" | "warning" | "error" | "info"

  - Five notification types with automatic icons
  - Custom duration and action buttons
  - Top-right positioning, theme-aware

  }
  ```

  ## Icon Standards

  ### Tabler Icons Usage
  Import icons from `@tabler/icons-react`:
  ```tsx
  import { IconChevronDown, IconX, IconCheck, IconLoader } from "@tabler/icons-react"
  ```

  **Common icons used:**
  - Navigation: `IconChevronDown`, `IconChevronLeft`, `IconChevronRight`
  - Actions: `IconX` (close), `IconCheck` (confirm), `IconMinus`

  - Default: `w-4 h-4` (16px) for inline elements
  - Large: `w-6 h-6` (24px) for header icons
  - Buttons: Match text size

  ```tsx
  <IconChevronDown className="w-4 h-4 text-muted-foreground" />
  ```

  ## Authentication Pattern

  ### NextAuth Configuration
  Location: `/lib/auth.ts`
  - CredentialsProvider for email/password
  - Token refresh mechanism with internal API calls
  - Session includes: `user.id`, `user.email`, `user.username`, `user.isVerfied`
  - AccessToken and refreshToken handled in session callbacks

  ### Authentication Hooks
  Location: `/hooks/use-auth.ts`
  - `useCurrentUser()` - Get current user or null
  - `useIsAuthenticated()` - Boolean check
  - `useUserInfo()` - Full user session data
  - `useRequireAuth()` - Redirect to signin if not authenticated

  ### Protected Routes
  - Auth routes: `/app/(auth)/signin`, `/app/(auth)/signup`
  - Protected: `/dashboard`, `/profile`, `/settings`, `/sessions`
  - API routes: `/api/auth/*`, `/api/users/*`

  ## Development Guidelines

  ### Creating Components
  1. Create component in appropriate folder (`/components/ui/`, `/components/forms/`, etc.)
  2. Use "use client" for interactive components
  3. Apply Tailwind classes with `cn()` for conditionals
  4. Use Tabler Icons for consistency
  5. Follow TypeScript strict mode
  6. Export as named export

  ### Example Component
  ```tsx
  "use client"

  import { Button } from "@/components/ui/button"
  import { IconChevronRight } from "@tabler/icons-react"
  import { cn } from "@/lib/utils"

  interface MyComponentProps {
    variant?: "default" | "outline"
    disabled?: boolean
  }

  export function MyComponent({ variant = "default", disabled }: MyComponentProps) {
    return (
      <Button 
        variant={variant} 
        disabled={disabled}
        className={cn("gap-2", disabled && "opacity-50")}
      >
        Click me
        <IconChevronRight className="w-4 h-4" />
      </Button>
    )
  }
  ```

  ### Form Components
  - Use HTML5 form elements (input, select, textarea)
  - Integrate with `/components/ui/Input`, `/components/ui/Label`
  - Add validation with client-side checks
  - Use API routes in `/api/` for server-side operations

  ### Pages & Layouts
  - Use app router patterns (folders for segments)
  - Group related routes with `(groupName)` syntax
  - Use `layout.tsx` for shared UI
  - Implement `useRequireAuth()` hook for protected pages
  - Load server session in page components when needed

  ## Naming Conventions
  - Files: kebab-case (my-component.tsx)
  - React components: PascalCase (MyComponent)
  - Functions/hooks: camelCase (useRequireAuth)
  - Constants: UPPER_SNAKE_CASE (API_URL)
  - Classnames: kebab-case utilities (bg-primary, text-foreground)

  ## Best Practices
  1. **Always use `cn()` for Tailwind conditionals** - Don't string concatenate classnames
  2. **Import components from relative paths** - Use `@/` for absolute imports
  3. **Keep components focused and composable** - Single responsibility principle
  4. **Use TypeScript types** - Define interfaces for props
  5. **Minimize client-side code** - Use server components when possible
  6. **Test responsive design** - Use Tailwind's responsive prefixes (md:, lg:, etc.)
  7. **Maintain color semantics** - Use semantic color names (primary, destructive, muted)
  8. **Follow dark mode support** - Use dark: prefix for dark theme styles
  9. **Document complex components** - Add JSDoc comments for reusable components
  10. **Keep BaseUI underneath** - Don't override BaseUI primitives directly; use Tailwind

  ## Common Tasks

  ### View Component Showcase
  Visit `/components-showcase` to see all custom components with their variants and usage examples in a live demo.

  ### Add a new UI component
  - Create in `/components/ui/component-name.tsx`
  - Base on shadcn/ui or create using BaseUI primitive
  - Style with Tailwind + `cn()` utility
  - Export named function component

  ### Add a new page
  - Create in `/app/route-name/page.tsx`
  - Use "use client" if interactive
  - Call `useRequireAuth()` if protected
  - Use Header component in layout if header needed

  ### Add a new API endpoint
  - Create in `/api/route-name/route.ts`
  - Handle GET/POST/PUT/DELETE methods
  - Validate input and authenticate session
  - Return JSON responses

  ### Add icons
  - Search Tabler Icons library: https://tabler-icons.io/
  - Import from `@tabler/icons-react`
  - Size: w-4 h-4 by default, adjust as needed
  - Color: use Tailwind color utilities (text-muted-foreground, text-destructive)
